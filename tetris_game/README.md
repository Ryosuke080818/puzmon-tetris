# テトリス風パズルゲーム(Windows PC 開発版)

Raspberry Pi + Raspberry Pi Pico + 各種センサー/LCD構成での動作を最終目標としつつ、
まずは Windows PC 上で pygame を使って開発するプロトタイプです。

## 実装済みの内容

- ホーム画面(「ゲーム開始」ボタン)
- ゲームプレイ画面
  - 標準的なテトリスのルール(7種一巡ランダム、回転、ライン消去)
  - 公式テトリスガイドライン準拠のスコア計算
    - シングル: 100×レベル
    - ダブル: 300×レベル
    - トリプル: 500×レベル
    - テトリス: 800×レベル
    - ソフトドロップ: 1点/マス、ハードドロップ: 2点/マス
    - 10ライン消去ごとにレベルアップ、レベルに応じて落下速度が上昇
  - スコア・レベル・ライン数・NEXTミノ・速度レベルを画面右側(画面端)に表示
- ゲームオーバー処理(「タイトルへ戻る」ボタン)
- 速度調整機能(暫定的にキーで代用)と、速度に応じたスコア倍率(×1.1刻み)

## 操作方法(Windows PC / キーボード)

| 操作 | キー | 備考 |
|---|---|---|
| 左移動 | ← | タクトスイッチ対応予定 |
| 右移動 | → | タクトスイッチ対応予定 |
| 下移動(ソフトドロップ) | ↓ | 長押しで加速、1マスごとに+1点 |
| 回転 | ↑ | タクトスイッチ対応予定 |
| 確定ボタン | Enter | プレイ中: ハードドロップ(一番下まで即落下、1マスごとに+2点) / ホーム・ゲームオーバー画面: 選択項目の決定 |
| メニュー選択(ホーム/ゲームオーバー) | ← → | ボタン選択の移動 |
| 速度レベル調整(暫定) | + / - | 将来的に可変抵抗のADC値読み取りに置き換え予定 |

## セットアップ

```bash
pip install -r requirements.txt
python main.py
```

## 日本語フォントについて

画面上の日本語表示のため、`game.py` の `load_japanese_font()` は
以下の優先順位でフォントを探します。

1. `assets/NotoSansJP-Regular.otf` (存在する場合)
2. OS標準の日本語フォント(Windows: Meiryo / MS Gothic 等)
3. デフォルトフォント(日本語が文字化けする可能性あり)

Windowsでは通常 2. で解決しますが、将来ラズパイ(Linux)へ移行する際は
日本語フォントが標準で入っていないことが多いため、
[Noto Sans JP](https://fonts.google.com/noto/specimen/Noto+Sans+JP) 等の
otf/ttfファイルを `assets/NotoSansJP-Regular.otf` として配置することを推奨します。

## 今後の実装予定・ラズパイ移行に向けたメモ

### 1. 可変抵抗による落下速度調整(本格実装)

現在は `+` / `-` キーで `ScoreManager.speed_level` (0〜9) を仮に操作していますが、
実機では Raspberry Pi Pico 側で可変抵抗をADC(アナログ入力)で読み取り、
0〜9 の段階値に変換して Raspberry Pi 側へシリアル通信等で送る想定です。

`input_handler.py` の `InputHandler.poll()` 内、`speed_up` / `speed_down` の
判定部分を、Pico から受信したADC値をそのまま `score_manager.set_speed_level(value)`
に渡す形に置き換えるだけで移行できるように、`game.py` 側は
「速度レベルという数値さえ受け取ればよい」設計にしてあります。

### 2. タクトスイッチへの入力置き換え

`input_handler.py` の `InputHandler` クラスが入力の唯一の窓口です。
キーボード読み取り部分(`pygame.key.get_pressed()` 等)を
GPIO(ラズパイ本体、または Pico 経由)のスイッチ状態読み取りに置き換えれば、
`game.py` 側のロジックは変更不要です。

### 3. LCD表示への置き換え

現在はPC画面の右側パネルにスコア等を表示していますが(画面端表示で代用)、
実機では小型LCD(SPI/I2C接続等)にスコア表示を行う想定です。
`game.py` の `_draw_side_panel()` がスコア表示のロジック部分にあたるため、
LCD描画用のライブラリ(例: luma.lcd 等)に応じた描画関数へ差し替える形になります。

### 4. 盤面・ボタン表示

現状 pygame ウィンドウ全体で描画していますが、実機では
- 盤面 → メインディスプレイ
- スコア等 → LCD
のように出力先が分かれる想定のため、`_draw_board_grid()` と
`_draw_side_panel()` を分離して実装してあります。
